# 第 8 集：生命周期、作用域和错误处理

## 本集目标

这一集讲 `vanilla-signal` 里负责“创建、销毁、清理、兜底”的 API：

- `createRoot`
- `createScope`
- `onCleanup` / `onDispose`
- `onMount`
- `getOwner`
- `catchError`
- `createErrorBoundary`

## 开场口播

前端代码不只是创建 UI，还要记得销毁 UI。

比如弹窗关闭后，里面的 effect、事件监听、定时器都应该释放。

如果只创建不清理，页面用久了就可能出现重复监听、重复请求、内存占用增加。

这一集我们学习生命周期和错误处理。

## 1. createRoot：创建一块可销毁的响应式区域

### 代码

```js
const { createRoot, createSignal, createEffect } = signal;

const dispose = createRoot((dispose) => {
  const [count, setCount] = createSignal(0);

  createEffect(() => {
    console.log('count:', count());
  });

  setCount(1);

  return dispose;
});

dispose();
```

### 口播

`createRoot` 会创建一个响应式根。

在这个根里面创建的 effect、memo、cleanup，都归这个 root 管。

调用 `dispose()` 后，这一整块响应式逻辑都会被销毁。

## 2. render 内部也是 root 思路

### 代码

```js
const dispose = render(() => jsx`<div>Hello</div>`, app);

dispose();
```

### 讲解

`render` 返回的也是销毁函数。

所以当你把 `vanilla-signal` 用在弹窗、抽屉、老项目局部增强时，要在这块 UI 不需要时调用 `dispose`。

## 3. onCleanup：清理资源

### 口播

只要你创建了浏览器不会自动帮你清掉的东西，就应该考虑 `onCleanup`。

比如定时器、事件监听、第三方组件实例。

### 代码

```js
const { createEffect, onCleanup } = signal;

createEffect(() => {
  const timer = setInterval(() => {
    console.log('tick');
  }, 1000);

  onCleanup(() => {
    clearInterval(timer);
  });
});
```

### 讲解

effect 重新执行前，会先执行上一次注册的 cleanup。

如果 root 被销毁，也会执行 cleanup。

## 4. 根据状态绑定事件，并自动清理

### 代码

```js
const [enabled, setEnabled] = createSignal(false);

createEffect(() => {
  if (!enabled()) return;

  const handler = (event) => {
    console.log('keydown:', event.key);
  };

  window.addEventListener('keydown', handler);

  onCleanup(() => {
    window.removeEventListener('keydown', handler);
  });
});
```

### 讲解

当 `enabled()` 从 true 变成 false 时，effect 会重跑。

重跑前，旧的 keydown 事件监听会被移除。

这就是响应式生命周期的价值。

## 5. onDispose：语义更明确的别名

### 口播

`onDispose` 是 `onCleanup` 的别名。

如果你的代码表达的是“销毁时清理”，用 `onDispose` 可读性会更好。

### 代码

```js
createRoot(() => {
  const widget = createThirdPartyWidget();

  onDispose(() => {
    widget.destroy();
  });
});
```

## 6. onMount：DOM 创建后的初始化

### 口播

有些逻辑需要等 DOM 插入后再执行。

比如自动聚焦输入框、测量元素、初始化第三方 UI 插件。

这时可以用 `onMount`。

### 代码

```js
let input;

render(
  () => {
    onMount(() => {
      input.focus();
    });

    return jsx`
      <input ref=${(el) => {
        input = el;
      }}>
    `;
  },
  app
);
```

### 讲解

`onMount` 会在当前同步代码结束后的微任务里执行。

通俗理解就是：先把 UI 建起来，再执行挂载逻辑。

## 7. createScope：工具层自己管理生命周期

### 口播

`createScope` 和 `createRoot` 类似，也能创建一块可销毁区域。

区别是它固定返回一个对象：`result`、`dispose`、`run`。

它更适合封装工具函数。

### 代码

```js
function createClock() {
  const scope = createScope(() => {
    const [now, setNow] = createSignal(new Date());

    const timer = setInterval(() => {
      setNow(new Date());
    }, 1000);

    onCleanup(() => clearInterval(timer));

    return now;
  });

  return {
    now: scope.result,
    dispose: scope.dispose,
  };
}

const clock = createClock();

createEffect(() => {
  console.log(clock.now());
});

clock.dispose();
```

### 讲解

这个 `createClock` 自己创建 signal、定时器和 cleanup。

调用方只需要拿到 `now` 和 `dispose`。

## 8. getOwner：了解当前上下文

### 口播

`getOwner` 会返回当前响应式 owner。

业务代码通常不需要直接使用它。

它更多用于调试、底层工具、框架集成。

### 代码

```js
createRoot(() => {
  console.log(getOwner());
});
```

### 讲解

初学者只要知道：`onCleanup` 这类生命周期 API 需要挂在当前 owner 上。

如果当前没有 owner，cleanup 就没有地方登记。

## 9. catchError：同步错误兜底

### 口播

如果你只是想安全执行一段同步代码，比如解析 JSON，可以用 `catchError`。

### 代码

```js
const { catchError } = signal;

const result = catchError(
  () => JSON.parse(text),
  (error) => {
    return {
      ok: false,
      message: error.message,
    };
  }
);
```

### 讲解

它适合同步函数。

如果是异步请求错误，还是应该用 `try...catch`、`createResource` 或 `createQuery` 的错误状态。

## 10. createErrorBoundary：响应式错误边界

### 口播

如果某块响应式逻辑可能抛错，但你不希望它直接影响外层，可以用 `createErrorBoundary`。

### 代码

```js
const { createErrorBoundary, createSignal, createEffect } = signal;

const [count, setCount] = createSignal(0);

const boundary = createErrorBoundary(() => {
  createEffect(() => {
    if (count() > 5) {
      throw new Error('count 太大了');
    }

    console.log('count:', count());
  });
});

createEffect(() => {
  if (boundary.hasError()) {
    console.error(boundary.error().message);
  }
});
```

### 讲解

`createErrorBoundary` 会创建一个边界。

边界内部的响应式错误会被捕获到 `boundary.error()`。

`boundary.hasError()` 可以判断是否已经出错。

`boundary.reset()` 可以重置边界。

`boundary.dispose()` 可以销毁边界。

## 11. 用错误边界做 UI 兜底

### 代码

```js
const boundary = createErrorBoundary(() => {
  render(
    () => jsx`
      <section>
        ${() => {
          if (state.count > 5) {
            throw new Error('数量不能超过 5');
          }

          return jsx`<p>${() => `数量：${state.count}`}</p>`;
        }}
      </section>
    `,
    app
  );
});

render(
  () => jsx`
    <section>
      ${() =>
        boundary.hasError()
          ? jsx`
              <div>
                <p>${() => boundary.error().message}</p>
                <button onClick=${() => boundary.reset()}>重试</button>
              </div>
            `
          : ''}
    </section>
  `,
  errorBox
);
```

### 讲解

这种写法更偏高级封装。

入门阶段不需要频繁使用错误边界，但当你封装复杂组件或插件时，它会很有用。

## 本集常见坑

### 坑一：创建了资源但没有销毁

弹窗、局部页面、临时组件关闭时，要调用 `dispose`。

### 坑二：事件监听没有 cleanup

只要手动 `addEventListener`，就要考虑对应的 `removeEventListener`。

### 坑三：把异步错误交给 catchError

`catchError` 只适合同步错误。请求错误更适合交给 `createResource`、`createQuery` 或 `try...catch`。

## 本集结尾

这一集我们学会了：

- `createRoot` 和 `createScope` 管理响应式作用域。
- `onCleanup` 和 `onDispose` 清理资源。
- `onMount` 在 DOM 创建后执行初始化。
- `catchError` 处理同步错误。
- `createErrorBoundary` 捕获响应式错误。

这些 API 是写可维护 UI 的基础。代码不只是能跑，还要能正确销毁、正确兜底。
